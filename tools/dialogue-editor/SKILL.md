# Dialogue Editor Tool — How It Works

## Overview

The dialogue editor is a **standalone HTML/JS/CSS application** — no Phaser, no build step, no npm dependencies. It lives at `tools/dialogue-editor/` and is served by Vite's dev server alongside the game.

**Three files:**
- `index.html` — HTML structure (toolbar, node list, canvas container, editor panel, preview)
- `editor.css` — Dark theme, three-panel layout, form styling
- `editor.js` — All logic: state, canvas rendering, event binding, data loading, export

## Loading the Editor

1. Run `npm run dev` in the project root
2. Open `http://localhost:3000/tools/dialogue-editor/`
3. The editor fetches `game.json`, `characters.json`, `variables.json` from `/data/`
4. Loads the first scene from the scene list and renders it

## Architecture

### State Management

A single `state` object holds everything:

```js
let state = {
  gameConfig: null,          // from game.json
  characters: {},            // from characters.json
  variableDefs: {},          // from variables.json
  sceneId: 'sample',         // current scene ID
  sceneData: null,           // raw scene JSON
  nodes: [],                 // current scene's nodes (with x,y added)
  selectedNodeId: null,      // for the editor panel
  camera: { x: 0, y: 0 },   // canvas pan offset
  zoom: 1,                   // canvas zoom level
  dragging: null,            // node drag state
  connectionDraft: null,     // wire-from-port state
  dirty: false,              // unsaved changes flag
  previewOpen: false,         // preview panel visibility
  allNodeIds: new Set()      // for duplicate ID checking
};
```

### Event Binding

Static buttons (in the HTML) are bound via `addEventListener()` in the `bindUI()` function, called on `DOMContentLoaded`. These use `document.getElementById()` to reference elements.

Dynamic buttons and inputs (generated by the editor form templates) use `onclick="window.__fn()"` and `onchange="window.__fn(this.value)"` attributes. The corresponding functions are explicitly attached to `window`:

```js
window.__updateField = (field, value) => { ... };
window.__updateType = (newType) => { ... };
window.__addChoice = () => { ... };
window.__choiceRow = (index, choice) => { ... };
// etc.
```

This is necessary because Vite's dev server processes inline scripts in a module scope, making top-level function declarations invisible to HTML `onclick` attributes.

### Canvas Rendering

[!IMPORTANT] The canvas render loop catches all errors silently to prevent crashes. If a rendering bug occurs, check `renderCanvas()` for issues.

The canvas is rendered via `requestAnimationFrame(renderLoop)`. Each frame:
1. Clear canvas
2. Draw grid (40px spacing, scaled by zoom)
3. Collect connections from node data (next, else, choices[].next)
4. Draw connection bezier curves with arrowheads
5. Draw each node as a rounded rectangle with:
   - Colored top bar (by type)
   - Node ID label
   - Type badge
   - Text/choice preview snippet
   - Input port (left circle)
   - Output ports (right circles, type-specific)
6. Draw connection draft line when user is dragging a wire

Coordinate system:
- World coordinates: the node positions (stored in JSON as x, y)
- Screen coordinates: canvas pixel positions
- Camera offset + zoom transforms between them

### Wire Connections

Users connect nodes by:
1. Clicking and dragging from an output port (right side circle)
2. A dashed line follows the cursor
3. Releasing near another node's input port (left side circle) creates the connection
4. The connection updates the source node's `next`, `else`, or `choices[].next` field

### Editor Forms

When a node is selected, the right panel renders a dynamic HTML form. The form content varies by node type:
- **Dialogue:** Speaker dropdown, expression text, text textarea, auto-advance toggle, next node selector
- **Choice:** Prompt text, add/remove choices with text/next/condition/variable-setting per option
- **Condition:** Variable picker, operator dropdown, value field, TRUE/FALSE target selectors
- **Event:** Event type dropdown, value field, next node selector
- **Wait:** Duration field, next node selector
- **End:** Ending text, next scene selector

All types have a "Set variable" section at the bottom for variable actions.

### Preview Panel

When toggled open, the preview panel renders the selected node as it would appear in-game:
- Dialogue: Shows speaker nameplate (with character color) and text with blinking cursor
- Choice: Shows prompt and numbered choice list
- End: Shows scene end text
- Other types: Shows type badge and next target

### Export

The Export button:
1. Serializes the current scene state to JSON
2. Cleans up empty/null fields
3. Preserves x, y coordinates
4. Creates a downloadable `.json` file via Blob + URL.createObjectURL

### localStorage Auto-Save

Every edit saves a draft to `localStorage['dialogue_editor_draft']`. On page load, the editor checks for a draft and offers to restore it. Draft includes: sceneId, nodes, gameConfig, characters, variableDefs.

## Keyboard Shortcuts

| Key | Action |
|-----|--------|
| Ctrl+S | Export scene |
| Ctrl+N | Add new node |
| Delete/Backspace | Delete selected node (with confirmation, only when not focused in an input) |
| Escape | Deselect node |

## Known Limitations

- **No file overwrite:** Export creates a download. User must manually place the file in `data/scenes/` and update game.json's scene list. The editor can't write to the filesystem.
- **No undo/redo for canvas operations:** Move is immediately applied. Undo only via localStorage draft restore.
- **No minimap:** The minimap div is present in CSS but not wired up.
- **Single selection only:** Only one node can be selected at a time. No multi-select or group operations.
- **No zoom-to-extents on initial load:** Camera starts at (0,0) zoom 1. Click "Fit" button after loading to center all nodes.

## Adding Features to the Editor

1. Add any new static buttons to `index.html` with IDs
2. Bind events in `bindUI()` in `editor.js`
3. If the feature involves new form fields, add them to the template strings in `renderEditor()`
4. Expose any callback functions on `window.__` for dynamic HTML access
5. Add the feature to `exportScene()` if it needs serialization
