# Phaser-NGE Memory & Decision Log

- *Gotcha:* The V2 editor backend relies on Vite dev-server plugins (`tools/editor-backend.js`) for things like `save` and `upload-asset`. This means right now it is coupled to Vite.
- *Decision:* Centralized backend API calls into `tools/shared/backend-adapter.js` to decouple the UI from raw `fetch` calls, preparing for an eventual Electron/desktop migration where IPC will be used.
- *Decision:* Replaced the hardcoded context menu in the visual node graph with a search palette (`Cmd+Space`) for better UX when adding nodes.
- *Gotcha:* The engine's starting scene is statically configured in `data/game.json` (`startScene`).
- *Decision:* To implement "Play from here", we write a debug target `{"sceneId", "nodeId"}` into `localStorage` (`nge_debug_start`) before opening a new window. `BootScene` intercepts this, parses it, and launches directly into `GameScene` with the debug parameters, bypassing the normal menu loop.
- *Gotcha:* Editor V2 `inspector.js` had incomplete DOM handling for `choice` nodes (it was just hardcoded to `...` for the choice list). We had to port choice list management (adding/removing branches, updating text/conditions/weights) over from the legacy standalone dialogue editor.
- *Gotcha:* Background rendering bug — `SceneController.onSceneStart` was not passing `layers` to GameScene. Scene data uses a `layers` array (new system) while legacy `background` field can be null. Without forwarding `layers`, `loadSceneLayers(undefined, null)` fell back to a gradient. Fix: add `layers: scene.layers` to both `onSceneStart` call sites in `SceneController.js`.

- *Gotcha:* Vite intercepts 404s and serves `index.html`. If the engine fetches a missing JSON file (like a deleted scene that is still listed in `game.json`), `r.ok` is still `true`, but parsing `r.json()` throws a SyntaxError (`Unexpected token '<'`).
- *Decision:* Wrapped the `fetch` calls in `BootScene` with a `safeFetchJson` that checks if the response text starts with `<` (HTML) and gracefully ignores it rather than crashing the game.
- *Decision:* Converted hardcoded Menu and Splash scenes into data-driven UI layouts powered by `data/theme.json` (`ui.menu` and `ui.splash`). Created visual drag-and-drop editors for these, providing a scalable foundation for a generic UI Builder.
- *Gotcha:* `GameScene.onAction` shows a ⚡ toast for ALL event types, including `bgm`/`sfx`. This was confusing — users saw the filename on-screen and thought that was the audio playing. Fix: suppress toast for `bgm`, `sfx`, `bgm_stop` event types (they are silent triggers).
- *Gotcha:* The event inspector used a plain text input for `eventValue`, so typing `bgm_imacomposernow` would not work if the audio file was missing from `public/assets/audio/bgm/`. The fix adds a dropdown hydrated from `/api/list-assets` so only real on-disk assets appear.
- *Decision:* Event node inspector now has context-aware fields: BGM/SFX → asset dropdown + volume slider; bg_change → background dropdown; camera → value input. Switching event type dispatches `inspector:refresh` to re-render without losing state.
- *Gotcha:* `eventVolume` is a new optional field on event nodes. It's passed through `SceneController.fireEvent → onAction({ volume })` and applied in `GameScene` before calling `playBGM`/`playSFX`. Default (null) = system volume unchanged.
- *Gotcha (ROOT CAUSE — audio cache miss):* `BootScene` previously checked `localStorage.getItem('nge_editor_data')` first. If stale data was present (written by the legacy standalone editor), it used that instead of disk files — missing any audio event nodes added since the last legacy export. The V2 editor (`state.js`) clears this key on save, but any browser session that pre-dated V2 could leave it around. **Fix:** Removed the `nge_editor_data` branch from `BootScene` entirely. V2 saves to disk only.
- *Decision:* Added `GameScene._loadAndPlay(type, key, onReady)` as a belt-and-suspenders runtime fallback. If a `bgm`/`sfx` event fires and the key isn't in Phaser's audio cache, it HEAD-probes extensions and loads the file on-the-fly via `this.load.audio` + `this.load.start()`. This means audio will play even if BootScene somehow missed a file (with a brief initial delay on first play).

- *Decision:* Removed rigid asset categories in favor of a free-form file system inside `/public/assets`. The editor now provides a dual-pane file explorer (`tools/views/assets.js`) to manage folders and files natively. Engine systems (`BootScene`, `LayerSystem`, `AudioSystem`, `CharacterSystem`) now expect exact relative paths (e.g., `backgrounds/city.png`) instead of relying on legacy prefixes (`bg_city`).
- *Gotcha:* In the `app.js` drop handler, injecting a call to an undefined `renderUI()` function caused a silent crash, preventing `renderOutline()` and `renderScenePreview()` from executing. This manifested as dropped assets not appearing until a manual browser refresh. Always explicitly call the correct modular render functions (`renderScenePreview`, `renderOutline`, `renderInspector`).
- *Decision:* Implemented interactive Transform Gizmos (Move, Scale, Rotate) directly inside the `canvas-viewport` by attaching mouse event listeners. We calculate deltas relative to the viewport zoom and pan to cleanly adjust layer properties (`x`, `y`, `scaleX`, `rotation`) in real time.
- *Decision:* Built a dedicated Keyframe Animation Editor tab (`data/animations/*.json`). Rather than booting a headless Phaser instance to preview tweens, we perfectly simulate Phaser's `tweens.chain` logic via a pure HTML/CSS DOM loop (`requestAnimationFrame` + CSS `transform`). This allows for lightning-fast, 1:1 previews of `x, y, scale, rotation, alpha` tweens.
- *Gotcha:* When performing JavaScript interpolations (e.g., `parseFloat(val) || defaultValue`), a parsed value of `0` acts as a falsy condition. This caused valid `0` values (for `x`, `y`, `rotation`) to improperly fallback to their default values (e.g. falling back to `1` for `scale`), preventing the CSS preview from updating correctly. Fix: Use an explicit `isNaN(parseFloat(val))` check instead of `||` logical OR.
- *Gotcha:* The native drag-and-drop `dataTransfer` payload from the Asset Browser only contains the local asset path (e.g., `characters/dave.png`). When assigning this to a `background-image: url(...)` in the Animation Editor, the browser resolves it relative to the `/tools/` directory, breaking the image. Fix: Manually prefix dragged asset paths with `/assets/` before applying them to DOM styles.
- *Decision:* Implemented `SceneController` node decoupling using `Registry.js` to allow nodes to define their own execution logic without polluting the central switch statement.
- *Gotcha:* Vite dev server `import` analysis fails on certain template literals; must construct paths with pure string concatenation when using dynamic imports or fetching data.
- *Gotcha:* `renderScenePreview` inline CSS transform is updated during gizmo drag for 60fps continuous dragging. Must also manually select `input` DOM elements to dispatch value updates for Inspector synchronization.
- *Decision:* Implemented Asset Drag-and-Drop in file explorer using HTML5 Drag and Drop API with custom `scene-drop-overlay` on the canvas, extracting `e.dataTransfer` for integration.
- *Gotcha:* In `app.js`, `this.layers.layers.has()` was used previously but `LayerSystem.js` stores layers in a plain Javascript Object `{}`, not a Map. Fix: changed access to `this.layers.layers[targetId]`.
- *Decision:* Implemented a stack-based scoping architecture for `VariableSystem.js`. Variables are localized to the current `call_scene` or `macro` node context and popped off when the sub-scene ends. Global context is maintained at index 0.
- *Decision:* Layer structure in `LayerSystem.js` and `scene-composer.js` is now recursive, allowing true Phaser Containers to nest child objects, built via HTML5 drag-and-drop in the Outline view.
- *Gotcha:* `play_animation` now seamlessly targets `Phaser.GameObjects.Container` IDs to tween multiple layer children natively, without any changes needed to the actual animation data.
- *Gotcha:* Drag-and-drop layer creation in `scene-composer.js` omitted the `type: 'background'` field. `LayerSystem.js`'s fallback gradient rendering added a blank gradient covering all layers because it thought no background was provided and `add()` placed it on top. Fix: Sent fallback gradient to the back of the container `this.container.sendToBack(graphics)`.
- *Gotcha:* Audio path duplication occurred because `BootScene` fetched using relative paths from `/assets/`, but `GameScene._loadAndPlay` prepended `audio/bgm/` again to already fully-resolved paths. Fix: Standardized path resolution in `GameScene` to match `BootScene` logic.

- *Decision:* Placed the `.desktop` launcher files in both `~/.local/share/applications/` (for application menu listing) and `~/Desktop/` (for easy desktop access) to seamlessly integrate the standalone Linux build of Phaser NGE.
- *Gotcha:* Desktop environments on Linux (such as GNOME or KDE) often require desktop launcher files and the application binaries to be explicitly marked as executable (`chmod +x`) before they can be trusted or run.
- *Gotcha:* The dialogue preview in `tools/app.js` was previously hardcoding the portrait image inside the dialogue box on the left, completely ignoring the `node.position` property (which supports left, center-left, center, center-right, right). It was also resolving character portraits from a non-existent `/assets/portraits/` folder instead of `/assets/characters/` (or the paths defined in `data/characters.json`).
- *Decision:* Updated `tools/app.js` to correctly resolve portrait paths using the character's portrait mapping (falling back to `/assets/characters/`) and extracted the portrait rendering from the dialogue text box into a standalone positioned element on the editor's preview canvas, matching the game's actual rendering behaviour.
- *Decision:* Implemented a "Recent Projects" dashboard for the Electron launcher. Stored as `recent-projects.json` in Electron's `app.getPath('userData')` directory. Added IPC endpoints `projects:getRecent`, `projects:addRecent`, `projects:removeRecent`, and `projects:setRoot` to let the frontend list, add, remove, and open directories directly. Redesigned the launcher UI with a premium dark-themed card layout, showing a split list of recent projects with relative time metadata ("opened 2h ago") and direct folder removal options.

- *Decision:* Configured `electron-builder` to bundle the application icon and a custom desktop entry in all Linux packages (RPM, DEB, and Linux Unpacked) by setting `"linux.icon": "icon.png"` and declaring them under `"extraFiles"`.
- *Gotcha:* In `electron-builder`, any desktop shortcut property overrides must be nested under `"linux.desktop.entry"` instead of directly in `"linux.desktop"`. Solved system-wide menu icon resolution for RPM/DEB by overriding the Icon location to `/opt/Phaser NGE/icon.png`.
- *Decision:* Simplified the app's title header from "Phaser-NGE — Editor v2" to "Phaser-NGE" by editing the `<title>` tag in `tools/index.html`.

- *Gotcha:* The `DialogueSystem` tried to access layers using `.has()` and `.get()` but `layers.layers` is a plain Javascript Object (`{}`), not a Map. Same issue was present for `characters.activeSprites` which was changed to `characters.portraits`.
- *Decision:* Replaced the `.find()` O(n) node lookup in `SceneController.jumpToId()` with an O(1) Map created in `startScene()` and rebuilt in `endScene()` when returning from the call stack.
- *Gotcha:* `play_animation` events failed because they looked up animation data via `this.sys.game.scene.keys.BootScene.Data` which does not exist, instead of importing the `Data` singleton.
- *Decision:* Removed `bg_` prefix on textures in `MenuScene` and `SplashScene`, matching `BootScene` caching strategy (raw paths). Cleaned up `theme.json` to have relative paths.
- *Gotcha:* `variables.json` contained type mismatches where variables of type 'number' or 'boolean' had default string values, causing bugs in logic evaluation. Fixed by matching default values to their declared types.
- *Gotcha:* The drag-and-drop `dataTransfer` payload from the Asset Browser contains a JSON string with `{path, type}` properties. Legacy Menu and Splash editor drop handlers only checked for plain text strings. Fix: Unified all drop handlers (Scene Composer, Menu, Splash) to correctly parse `application/json` and `text/plain` payloads to extract the relative file `path`.

- *Decision:* Voice Acting implemented via a dedicated audio channel. Implemented a 200ms quick fade out in \AudioSystem.js\ to avoid clicking sounds when the player rapidly skips through voiced dialogue lines.
- *Decision:* Implemented \GlobalSaveSystem\ capabilities in \SaveSystem.js\ to track persistent data like unlocked CGs (and eventually achievements) via the \
arrative_globals\ localStorage key.
- *Gotcha:* The CG Gallery UI needs to know the total number of gallery images to display locked \?\ placeholders, but we have no backend at runtime. *Fix:* Intercepted the \/api/save\ request in the editor backend to automatically scan \public/assets/gallery/\ and compile a master list into \game.gallery\ upon save.

- *Decision:* Blocked browser history navigation (back/forward mouse buttons) inside the editor V2 SPA to prevent accidental exits. Used a combination of `mousedown` preventDefault (for buttons 3 and 4) and a dummy history pushState/popstate trap.
